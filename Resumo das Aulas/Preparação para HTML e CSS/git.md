# GIT

Git é um sistema de controle de versão distribuído que permite aos desenvolvedores rastrear as mudanças em seu código fonte ao longo do tempo <br>
e colaborar com outros desenvolvedores em um projeto. Ele é amplamente utilizado na indústria de software para gerenciar projetos de código aberto e fechado.<br>

GitHub é uma plataforma de hospedagem de código fonte que utiliza o Git como seu sistema de controle de versão. Ele permite que os desenvolvedores <br>
armazenem e gerenciem seus projetos Git em repositórios online, além de fornecer ferramentas para colaboração, como controle de acesso, revisão de código<br> 
e gerenciamento de problemas.

A principal diferença entre Git e GitHub é que o Git é um sistema de controle de versão, enquanto o GitHub é uma plataforma de hospedagem de código fonte.<br> 
O Git é um software que você instala no seu computador local, enquanto o GitHub é uma plataforma na nuvem que permite que você hospede seus repositórios <br>
Git online e colabore com outros desenvolvedores.

Resumindo, o Git é uma ferramenta de controle de versão que permite que você rastreie as mudanças em seu código fonte ao longo do tempo, enquanto o GitHub <br>
é uma plataforma online que utiliza o Git para armazenar e gerenciar projetos de código fonte, além de fornecer ferramentas de colaboração para desenvolvedores.
